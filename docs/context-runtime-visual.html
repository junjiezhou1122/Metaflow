Personal Context Runtime
Raw context → Views → Memory → Plugins
Design Philosophy / 浅色版

原始 context 是事实源，其他一切都是 view。

我们不是在做一个 AI 笔记本，而是在做一个本地优先的 personal context substrate：自动捕捉真实工作流，把事实保存下来，再让不同插件按自己的目的组织、压缩和行动。

Raw context is source of truth. Everything else is a view. Memory is a durable, purpose-specific compressed view. Plugins define how to attend, compile, and act.

不是一个中心 assistant

Paperboy 的启发是“AI 应该进入真实上下文”。我们的推进是：不要让一个 assistant 统治所有信息，而是构建可编程的 context runtime。

Screenpipe 的位置

  • Screenpipe captures
    屏幕、OCR、accessibility、音频、窗口、URL、输入事件。
  • Runtime organizes
    解析 workspace、生成 views、维护 evidence graph。
  • Plugins create value
    语言学习、coding memory、research、daily summary。

整体链路

核心不是“Raw → Thread → Plugin”，而是“Raw Observation Store → 多种 ContextView → Context Broker → Plugin”。Thread 只是其中一种 view。

01 / CAPTURE

Raw Observation

Screenpipe、Browser、Git、AI sessions、notes、files、manual save。

02 / STORE

ContextRecord

统一 envelope：source、time、content、payload、privacy、provenance。

03 / VIEWS

Context Views

Timeline、Workspace、WorkThread、Episode、Entity、Semantic、MemoryView。

04 / PLUGINS

Broker + Plugins

按 attention policy 取 context，编译 memory，产生 suggestion / action。

三层概念

所有上层结构都来自 observation，但它们的损耗、稳定性和用途不同。

SOURCE OF TRUTH

Observation

被捕捉到的事实。它只回答：发生了什么、何时发生、来自哪里、权限是什么。

ORGANIZATION

ContextView

对 observations 的整理、聚类、索引或投影。WorkThread、Timeline、Workspace 都是 view。

DURABLE VIEW

Memory

一种特殊 view：面向目的、可长期复用、允许有损压缩，并会影响未来行为。

All memories are views. Not all views are memories.

Memory 不是神秘实体，它是 durable / purpose-specific / compressed view。
Timeline View按时间组织 observation，用于回放、daily summary、时间线浏览。
Workspace View判断当前项目路径，例如 `/Users/junjie/info`，用于关联 git / files / AI sessions。
WorkThread View任务连续性视图，适合 coding handoff、agent continue、daily summary。
Plugin MemoryView语言学习、coding、research 等插件各自编译的长期可复用压缩 view。

Thread 不是宇宙中心

Thread 很适合 coding，但不应该强迫所有插件都经过 thread。插件可以使用 thread，也可以跳过 thread 直接查询 raw observations / views。

Thread 很有用的场景

Coding memory / agent handoff / daily summary：需要知道某段时间围绕哪个项目、哪些文件、哪些 AI session、哪些屏幕证据在推进。

Thread 不一定需要的场景

Language learning：更关心最近 7 天真实接触的英文材料、词频、语境、例句。它可以直接读 browser/screen/AI chat text exposure。

Plugin = Attention + Compiler + Action

Plugin 不是 prompt。它声明自己关注什么、如何编译 view/memory、能做什么动作、需要什么权限。

Attention Policy

定义插件要看哪些 source、schema、时间范围、workspace、thread、semantic query 或 memory views。

Memory / View Compiler

把 observations/views 编译成插件自己的 durable views，例如 vocabulary exposure 或 project decisions。

Permissioned Action

只在权限允许范围内 suggest、draft、local execute 或 external act，并保留 provenance。

Context Broker

插件可以跳过 thread，但不能跳过 broker。Broker 负责权限、范围、证据、token budget 和来源追溯。

{
  "plugin_id": "language-learning",
  "mode": "source",
  "time_window": "7d",
  "schemas": [
    "observation.browser_page_snapshot",
    "observation.screenpipe_activity",
    "observation.ai_chat"
  ],
  "privacy": "private"
}

  }
}

Interpreter 的真实位置

Thread Interpreter 不是 plugin，也不是 memory compiler。它只是 WorkThread View 的 display compiler。

组件它回答的问题当前设计边界
Workspace Resolver用户现在大概率在哪个项目/文件夹工作？deterministic first，使用 window/cwd/path/git/Screenpipe signals。
WorkThread View哪些 records 可能属于同一段任务连续性？evidence index，不是事实源，不是所有 plugin 的必经入口。
Thread Interpreter这个 thread 如何人类可读地显示？只做 display_title / brief / confidence；暂时不依赖 tags 和 next_steps。
Plugin Compiler基于某个目的，应该压缩出什么长期可复用 view？每个 plugin 自己定义，例如 language memory / coding memory。

从同一批 observation 编译不同 memory

这就是“plugins attend differently”的含义：同一段真实工作流，可以服务不同应用。

Coding Memory

当前项目在实现 Screenpipe workspace resolver；关键文件是 `src/runtime.ts`、`src/screenpipe.ts`；设计原则是 thread optional。

Language Memory

最近高频 exposure：provenance、substrate、resolver、ambient、retention。可以生成技术语境的个性化例句。

Research Memory

用户在研究 Paperboy、Screenpipe、personal AI、plugin ecosystem，可形成 topic map 和阅读队列。

设计原则

这些原则比任何单个实现都重要，后续 schema、connector、plugin 都从这里生长。

01Capture facts, don't judge too early.
先捕捉事实，不急着总结成 memory。
02Raw context is the source of truth.
任何 view / memory 都必须能追溯回 raw evidence。
03Everything above raw context is a view.
Thread、timeline、entity、semantic、memory 都是不同 view。
04Memory is durable compressed view.
Memory 是可长期复用、面向目的、允许有损压缩的 view。
05Plugins choose their own attention.
Plugin 可以用 thread，也可以跳过 thread。
06Provenance and permission are mandatory.
插件访问 context 和执行动作必须有来源和权限边界。
